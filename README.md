# URL Shortener

Веб-застосунок для скорочення посилань зі статистикою переходів. Реалізований як тестове завдання на базі **Symfony 8.1**.

---

## Функціональність

### Скорочення посилань
- Користувач вводить оригінальний URL та необов'язковий термін дії (у днях)
- Система генерує унікальний 6-символьний короткий код із алфавіту `[0-9a-zA-Z]`
- За коротким кодом відбувається редирект на оригінальний URL
- Якщо термін дії посилання вийшов - повертається сторінка з HTTP 410 Gone

### Статистика переходів
Для кожного короткого посилання доступна сторінка `/statistic/{code}` з такими даними:

| Показник | Опис |
|---|---|
| **Total clicks** | Загальна кількість переходів |
| **Unique clicks** | Кількість переходів з унікальних IP-адрес |
| **Clicks by day** | Графік кількості переходів по датах |
| **Referrers** | Звідки прийшли користувачі (заголовок `Referer`). Прямі переходи відображаються як *Direct* |
| **Browsers** | Розподіл переходів за браузерами (Chrome, Firefox, Safari тощо) |
| **Devices** | Розподіл переходів за типом пристрою (desktop, mobile, tablet) |

Для кожного переходу фіксуються: IP-адреса, User-Agent, Referer, браузер та тип пристрою.

## Структура БД

**`urls`** — скорочені посилання
- `id` — ULID (primary key)
- `original` — оригінальний URL
- `short_code` — унікальний 6-символьний код
- `created_at` — дата створення
- `expires_at` — дата закінчення дії (nullable)

**`clicks`** — переходи за посиланням
- `id` — ULID (primary key)
- `url_id` — зовнішній ключ на `urls`
- `ip` — IP-адреса відвідувача
- `user_agent` — рядок User-Agent
- `referer` — заголовок Referer
- `browser` — назва браузера
- `device` — тип пристрою
- `created_at` — дата переходу

## Запуск проєкту

### Вимоги
- Docker + Docker Compose
- Make

### Перший запуск

```bash
make init
```

Команда виконає: збірку контейнерів → запуск → встановлення залежностей через Composer → міграції БД.

### Доступні команди

```bash
make up              # Запустити контейнери
make down            # Зупинити контейнери
make logs            # Переглянути логи
make migrate         # Запустити міграції
make composer-install           # Встановити залежності
make composer-require package=vendor/name  # Додати пакет
make php             # Відкрити bash у PHP-контейнері
```

Застосунок доступний за адресою: **http://localhost:8000**

## Тести

Проєкт покритий **feature** та **unit** тестами на базі PHPUnit 13.

### Структура тестів

```
tests/
├── Controller/
│   ├── UrlControllerTest.php         # Тести ендпоінтів скорочення та редиректу
│   └── StatisticControllerTest.php   # Тести сторінки статистики
├── Repository/
│   └── ClickRepositoryTest.php       # Тести запитів статистики до БД
└── Service/
    └── CodeGeneratorTest.php         # Юніт-тести генератора коротких кодів
```

### Запуск тестів

```bash
# Перший раз — створити тестову БД
make test-setup

# Запустити всі тести
make test
```
